# 📊 Retail Sales Analysis using SQL & Power BI

## 📌 Project Overview
This project focuses on analyzing retail sales data using SQL and visualizing insights using Power BI. The objective is to extract meaningful business insights and present them through an interactive dashboard.

---

## 🎯 Objectives
- Analyze overall sales performance  
- Identify top-performing regions, categories, and products  
- Understand customer purchasing behavior  
- Analyze monthly sales trends  
- Present insights through an interactive dashboard  

---

## 🛠️ Tools & Technologies
- SQL (SQLite) — Data Analysis  
- Power BI — Data Visualization  
- DB Browser for SQLite  

---

## 📂 Dataset
- Superstore Sales Dataset (CSV format)  
- Contains order details, customer data, product information, and sales  

---

## 🔄 Project Workflow

### 1. Data Analysis using SQL
- Performed aggregation using `SUM()` to calculate total sales  
- Used `GROUP BY` to analyze sales by region, category, and state  
- Identified top customers and products using sorting and limiting  
- Analyzed monthly sales trends using date functions  

### 2. Data Visualization using Power BI
- Built an interactive dashboard to visualize key insights  
- Included KPIs, filters, and category-wise analysis  
- Enabled easy identification of trends and performance metrics  

---

## 📊 Key Analysis Performed
- Total Sales Calculation  
- Sales by Region and State  
- Category-wise Performance  
- Top Products and Customers  
- Monthly Sales Trend Analysis  

---

## 🔍 Key Insights
- Certain regions contribute significantly higher revenue compared to others  
- A small number of products generate a large portion of total sales  
- Customer spending is uneven, with top customers contributing heavily  
- Sales vary across months, indicating potential seasonal trends
